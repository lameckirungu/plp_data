# Week 2: Data Wrangling with Pandas & Operational Data Structures

## Overview

Week 2 was where I encountered reality. I worked with messy, real-world datasets and mastered the industry-standard tools for data cleaning and manipulation. This week transformed broken sensor logs into reliable analytical assets.

## What I Learned

I mastered Pandas and NumPy for data wrangling and manipulation. I ingested messy data—nulls, typos, wrong formats—and applied programmatic cleaning techniques. I aggregated and aligned time-series data, preparing datasets for analysis and visualization.

## Tools I Used

**Core Libraries**: pandas, numpy, matplotlib, seaborn (pre-installed in Anaconda)

**IDE**: VS Code and Jupyter Lab

**Datasets** (provided via PLP LMS):
- kpc_sensor_log_dirty.csv

- hospital_wait_times.json

**Collaboration**: GitHub (for daily cleaning scripts), Zoom (for stand-ups), Peer Pod Discord/Slack

**Optional**: missingno library for visualizing missing data patterns

## The Challenge I Faced

Real operational data contains:
- Null values and missing records
- Typos and data entry errors
- Inconsistent formatting and units
- Duplicates and misaligned timestamps
- Out-of-range or nonsensical values

## My Process

1. **Load & Inspect**: Read data and understood its structure and quality
2. **Clean & Standardize**: Handled nulls, fixed types, normalized values
3. **Transform & Align**: Aggregated by time-series, joined datasets, reshaped as needed
4. **Validate & Document**: Ensured cleaned data was ready for analysis

## Key Takeaway

By Friday, I transformed a broken sensor log into a reliable analytical asset. The missingno library was invaluable—empty spreadsheet rows became a literal visual map of where my data needed healing. That clarity turned a chaos of errors into an organized cleaning strategy.

**The result**: Clean, structured, actionable data ready for insight generation and visualization.
