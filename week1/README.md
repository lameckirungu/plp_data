# Week 1: Foundations of Operational Data Analytics

## Overview

Week 1 introduced the fundamentals of Operational Data Analytics (ODA), establishing both the conceptual framework and practical skills needed to work with data in real operational environments.

## Key Concepts

### The Mindset: Operational Data Analytics Loop

We defined ODA as a continuous cycle: Operations → Data → Insight → Action. This framework applies universally across domains—whether monitoring KPC pipeline pressure, hospital patient wait times, or retail inventory levels.

### The Toolkit

Successfully set up the development environment:
- Installed Anaconda
- Configured Jupyter Lab and VS Code
- Set up GitHub repositories for version control and collaboration

### The Syntax: Python Fundamentals

Moved beyond basic "Hello World" programs to write practical Python functions that:
- Calculate operational KPIs (Stock Turnover, Efficiency Ratios, etc.)
- Classify operational states (Normal/Warning/Critical)
- Handle real-world business logic

### The Structure: Data Organization

Learned to model operational entities using:
- Lists for sequential data
- Dictionaries for structured, key-value data
- JSON parsing for data interchange

### The Soft Skills

Developed professional collaboration and communication abilities:
- Formed Peer Pods for collaborative learning
- Delivered public introductions to cohort members
- Wrote Data Memos explaining technical concepts to non-technical audiences

## Reflection

As you move into Week 2's messier data, remember: can you explain the difference between a list and a dictionary to a depot manager without using code? This clarity of thought translates directly to clearer data analysis and better business communication.
